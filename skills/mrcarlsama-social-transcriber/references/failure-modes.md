# 失败类型

## 需要新鲜 Cookie

原因：provider 能解析视频 ID，但拿不到元数据或媒体地址。

处理：要求用户显式提供 Cookie 文件，或提供手动下载好的本地视频。不要自动读取浏览器 Cookie。

## 没有可下载媒体

原因：元数据能解析，但没有返回可播放的媒体地址。

处理：重试一次。如果仍然失败，告知用户视频可能被限制、删除、区域限制或平台保护。

## 音频抽取失败

原因：`【标题】原视频.<ext>` 存在，但 ffmpeg 无法抽取音频。

处理：检查 `【标题】原视频.<ext>` 大小，并运行 `uv run --script <skill_dir>/scripts/preflight.py --check`。文件可能已损坏，或根本不是有效视频。

## 本地 ASR 不可用

原因：`faster-whisper` 导入失败、模型下载失败，或 CPU 运行时失败。

处理：脚本会从指定模型开始自动向下重试。必要时先运行 `bootstrap.py --ensure` 和 `preflight.py --check`，再手动指定更小模型：

```bash
--model tiny
```

## 逐字稿为空

原因：视频没有人声、音频抽取静默失败、语言识别错误，或 ASR 没检测到语音。

处理：脚本会自动降级 ASR 模型重试。仍为空时，检查 `【标题】原音频.wav`，并确认视频确实有可听见的人声。

## 部分产物已存在

原因：上一次运行在下载、音频抽取或 ASR 中途失败。

处理：再次运行同一链接。脚本会复用已有原视频、原音频、逐字稿、字幕和 `words.json`，不会从头重做已经完成的步骤。
